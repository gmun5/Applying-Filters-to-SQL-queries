# Applying Filters to SQL queries

## Objective


In this scenario, I acted as a security professional analyzing organizational data to identify potential security concerns related to employee activity and login attempts. The objective was to query and filter data from structured datasets to retrieve relevant information for investigation.

### Skills Learned


- SQL data querying and filtering using SELECT and FROM
- Developed the ability to use conditional logic (WHERE, AND, OR, NOT) to isolate relevant information
- Learned how to analyze datasets to identify patterns such as failed login attempts and unusual activity
- Built familiarity with pattern matching using LIKE and wildcard operators (%) for flexible data searches
- Strengthened problem-solving skills by narrowing down large datasets into meaningful results
- Improved understanding of how data analysis supports security investigations and decision-making

### Tools Used


- SQL (Structured Query Language) for querying and filtering data
- Relational database tables
